## README/Product Synopsis

Mantis is an agentic coding system for bounded engineering work.

It takes a concrete task, routes to the cheapest safe execution path, isolates risky changes when needed, verifies the result, and exposes enough detail to trust what happened.

Core product traits:

- cheap routing instead of default premium execution
- isolated workers for multi-file tasks
- background jobs and approvals
- verifier-backed completion
- visible diffs, traces, and execution history

Best use cases:

- bounded fixes
- structured edits
- test generation with verification
- reviewable multi-worker runs

Current limitation:

Mantis is strongest when the task is concrete and checkable. It is not yet a general autonomous engineer for open-ended repo work.

## Launch Post Draft

Mantis is our attempt to make coding agents useful by making them narrower, cheaper, and more inspectable.

The pitch is simple: give Mantis a concrete engineering task and it should choose the cheapest safe path, isolate risky edits, verify the result, and show its work.

We did not build this as an “agent OS.” We built it for bounded coding work:

- read a repo
- make a change
- run or verify the right checks
- expose the diff and execution trail

What matters to us is not whether the model can produce an impressive demo answer. What matters is whether the system can complete real bounded tasks with evidence.

That is why Mantis is built around:

- cheap-by-default routing
- isolated workers
- visible approvals
- background execution
- verifier-backed completion

Today, Mantis is strongest on bounded engineering tasks, not open-ended autonomous software development. That is an intentional choice. We would rather be precise and useful than broad and untrustworthy.

If you want a coding agent that shows its work instead of hiding behind confidence, that is what Mantis is for.
