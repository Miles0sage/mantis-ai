# Positioning Copy

## Homepage Version

Mantis is an agentic coding product for bounded engineering work.

Give it a concrete task, and it routes to the cheapest safe path, isolates risky edits, verifies the result, and shows its work.

What it is good at now:

- bounded read, edit, fix, and refactor tasks
- background jobs with visible progress
- isolated workers for multi-file work
- verifier-backed completion
- inspectable diffs, traces, and decision history

What it is not:

- a general autonomous software company
- a magic agent for open-ended repo surgery
- a black box that asks for trust without evidence

The product wedge is practical:

- cheap-by-default routing
- visible approvals
- isolated worktrees
- benchmarked real tasks
- honest verification

Current proof:

- `233` tests passing
- scenario and stress benchmarks green
- curated live/provider benchmark at `20/20`

Mantis is strongest when the task is concrete, bounded, and verifiable.

## Investor Version

Mantis is building the execution layer for bounded coding work.

The market does not need another vague “agent OS” pitch. It needs a system that can take a concrete engineering task, choose a cheap safe path, isolate the work, verify the result, and expose enough detail to trust it.

That is the wedge:

- low-cost routing instead of defaulting to premium models
- worker isolation for risky multi-file changes
- verifier-backed completion instead of pure model self-report
- background execution and approvals for real workflows
- benchmarked task performance instead of demo theater

What is validated today:

- bounded read/edit/fix/refactor flows
- multi-worker isolation and reviewability
- server responsiveness during active jobs
- repeatable synthetic and live benchmark coverage

What is not overclaimed:

- open-ended autonomous repo surgery
- universal reliability on arbitrary software tasks
- solved model-latency problems

This should be understood as a productized coding operator for bounded work, not a general AGI engineer.

That is a better market position because it is true, defensible, and already supported by the runtime and benchmark evidence.

## Internal Version

Mantis is credible now, but only if we keep the positioning narrow.

The product is real when we describe it as:

- a bounded coding operator
- with cheap routing
- isolated workers
- verifier-backed completion
- visible diffs and traceability

The product becomes fake when we describe it as:

- a general autonomous engineer
- an agent OS
- a full software company in a box

Current strengths:

- bounded tasks work
- the benchmark story is now strong
- worker isolation and resume mechanics are real
- the system is materially better than it was at the start of this cycle

Current weak spots:

- model-backed latency
- some long-tail refactor/reporting behavior
- broad open-ended tasks still underproven

Rule for messaging:

- sell the verified wedge
- publish the benchmarks
- document the limits
- do not market the unproven path as solved

Best one-line description:

Mantis is the fastest credible open coding agent for bounded engineering work: read a repo, make a change, prove it, and show its work.
